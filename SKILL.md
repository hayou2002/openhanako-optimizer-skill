# OpenHanako 优化大师 v2.0

---

## 唯一入口：说「优化大师」

### 入口

你说「优化大师」，直接展示菜单。

### 菜单内容

```
⚙ 优化大师 — 请选择要优化的项目：

  ① 工作区   → 目录结构、项目规范、文件归位
  ② 缓存     → 日志/检查点/图片缓存
  ③ 技能     → 同名/废弃/bundle检测
  ④ 经验与记忆 → 经验提炼/去重/记忆整理
  ⑤ 会话     → 空目录/大文件
  ⑥ 插件     → 孤儿目录/缓存/补丁
  ⑦ 配置     → 设置合理性检查（只读）
  ⑧ 开发     → 开发流程/临时文件清理
  ⑨ 运行时   → 归档对话/缓存总容量
  ⑩ 初始化   → 首次使用：建立目录/文档/模板

  说「全部」一次扫描全部
  说「退出」结束
```

### 触发对照表

| 你说 | 我做什么 |
|------|---------|
| 「优化大师」 | 检测初始化 → 展示菜单 |
| 「初始化」/「⑩」 | 执行初始化流程 → 展示菜单 |
| 项目编号或名称 | 只扫该项目 → 报告 → 确认执行 → 回菜单 |
| 「全部」 | 依次扫所有项目 → 逐项确认执行 → 回菜单 |
| 「退出」 | 结束 |

## 初始化流程

> 说「初始化」或首次使用自动触发。只建不删。

### 第1步：建立标准目录结构

创建以下目录（尚不存在的）：

```
_temp/对话/{主题}/截图/ 数据/ 脚本/ 导出/
_temp/项目/
_temp/一次性/
_temp/工作区残留/
_temp/检查点/
_tools/脚本/{服务器管理/数据同步/备份/知识管理}/
_tools/模板/{思源笔记/考试备考}/
_tools/参考/
data/backups/
data/archives/
data/数据集/
projects/插件/ 技能/ 应用/ 数据/
个性化/资料/{备考/通用参考/个人文档}/
个性化/事项/
个性化/联动/{思源笔记/记忆助手-MemOS/移动云盘}/
个性化/专项/{习惯/技能/认知/领域}/
个性化/Hanako/{技能/认知/习惯}/
```

### 第2步：写入参考文档

- 写入 `_tools/参考/文件组织规范.md`（默认版本）
- 写入 `_tools/参考/优化规则手册.md`
- 写入 `_tools/参考/清理执行手册.md`

### 第3步：写入模板文件

创建以下文件（尚不存在的）：

```
个性化/资料/资料索引.md
个性化/事项/{事项名}-{状态}/事项索引.md（待使用时创建）
个性化/专项/{领域}/成长记录.md
个性化/Hanako/{领域}/README.md 或 空目录初始化
```

### 第4步：运行适配检测

检测当前环境，标记哪些个性化规则适用。

### 第5步：输出初始化报告

```
✓ 目录结构已建立
✓ 参考文档已写入
✓ 模板文件已创建
✓ 适配检测完成（启用 X 项个性化规则）

初始化完成。现在你可以选择要优化的项目了。
```

## 适配检测

每次扫描前自动检测当前环境。未检测到的跳过。

```
检测项                          → 启用
有 _tools/参考/优化规则手册.md    → 已初始化
有 个性化/资料/备考/             → 工作区备考规则
有 个性化/联动/思源笔记/         → 思源笔记联动
有 个性化/联动/记忆助手-MemOS/   → MemOS 规则
有 个性化/Hanako/               → Hanako 知识库
有 个性化/事项/{事项}/           → 事项三模协作
```

## 核心执行流程（单项目模式）

> 选一个项目，走四步。走完自动回到菜单。

### 阶段1：扫描

只扫你指定的那个板块。结果标记：✅ 正常 / ⚠️ 建议 / 🔴 需处理 / ⛔ 不适用

### 阶段2：输出报告

- 🔴 需要你确认 → 逐条列出
- ⚠️ 自动执行 → 标注"自动处理"
- ✅ 正常 → 带过

### 阶段3：确认执行

- 🔴 逐条等你确认（做/跳过）
- ⚠️ 自动执行
- ✅ 跳过

操作留痕 `_temp/检查点/`。

### 阶段4：沉淀

1. 经验提炼 → `Hanako/技能/`
2. 认知记录 → `Hanako/认知/`
3. 索引更新
4. 留痕追加

→ **自动回到菜单，等你选下一项。**
