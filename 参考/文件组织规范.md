# 文件组织规范

> 由 OpenHanako 优化大师维护。优化模式更新规则，清理模式按规则执行。

---

## 一、温度体系总览

| 文件夹 | 温度 | 定位 | 管理逻辑 |
|--------|------|------|---------|
| `_temp/` | 🔥 热 | 临时缓冲区 | 存→流转→删/移出，7~30天生命周期 |
| `_tools/` | ☀️ 暖 | 高频复用工具箱 | 严格准入，不自动过期，>90天闲置预警 |
| `projects/` | 🌤 常温 | 分版本项目管理 | 按类别+状态命名，版本文件夹+索引 |
| `个性化/` | ☀️ 暖 | 个人云盘+事项+知识库 | 资料/只进不出；事项/有生命周期；专项/只增不减 |
| `data/` | ❄️ 冷 | 冷区存储 | 从其他目录冷却进来，备份保留7份 |

## 二、`_temp/` 缓冲区管理规则

### 2.1 前置分流

```
文件产生
  ├─ 有明确归处？  有→直接进目标目录
  ├─ 文件 >50MB？  是→进 data/
  └─ 以上均否→进 _temp/（工具自动产生的临时文件例外）
```

### 2.2 生命周期

| 目录 | 保留期 | 超期动作 | 移入目标 |
|------|--------|---------|---------|
| `对话/{主题}/截图/` | 7天无新文件 | 建索引移入 | `个性化/资料/`下对应目录 |
| `对话/{主题}/数据/` | 14天无新文件 | 建索引移入 | `个性化/资料/`或`_tools/数据/` |
| `对话/{主题}/脚本/` | 14天无新文件 | 提示可复用→移入 | `_tools/脚本/{功能分类}/` |
| `对话/{主题}/导出/` | 30天无新文件 | 按内容归入 | `个性化/资料/`下对应目录 |
| `项目/{项目名}/` | ≥14天无改动 | 提示结项→移入 | `projects/{分类}/{项目名}-存档/` |
| `一次性/` | 创建≥7天 | 确认后删 | — |
| `工作区残留/` | 进入≥7天 | 确认后删 | — |
| `检查点/` | 保留3份 | 自动删旧 | — |

### 2.3 一次性内容规范

```
_temp/一次性/{YYYYMMDD}-{描述}/
├── 目录说明.md    ← 必建（用途/来源/文件清单/后续状态）
└── 文件...
```

### 2.4 索引更新

移入目标目录时立即生成索引；清理扫描时自动遍历重写所有索引。

## 三、`_tools/` 工具箱管理规则

### 3.1 目录结构

```
_tools/
├── 脚本/{功能分类}/  ← 服务器管理/数据同步/备份/知识管理
├── 模板/{用途}/      ← 思源笔记/考试备考
└── 参考/            ← 只读参考文档
```

### 3.2 准入标准

进 `_tools/` 必须满足：**已验证可用**、**有使用说明**、**≤50MB**。缺一不可。

### 3.3 移入路径

| 来源 | 移入 | 条件 |
|------|------|------|
| `_temp/对话/{主题}/脚本/` | `_tools/脚本/{功能分类}/` | 已复用或确定可复用 |
| `_temp/对话/{主题}/导出/` 模板类 | `_tools/模板/{用途}/` | 模板风格被确认 |
| 外部下载脚本 | 先放 `_temp/工作区残留/` 验证 | 验证通过后移入 |

### 3.4 生命周期

不自动删除。连续 90 天未使用 → 提示是否归档到 `data/archives/`。

### 3.5 索引

每个功能分类维护 `脚本索引.md`，清理扫描时自动更新。

## 四、`projects/` 项目管理规则

### 4.1 分类规则

```
projects/
├── 插件/    ← Hana 插件项目
├── 技能/    ← SKILL 技能包
├── 应用/    ← 独立软件/工具/服务
└── 数据/    ← 数据集/配置模板/非代码项目
```

### 4.2 命名规则

`{项目名称}-{项目状态}`

状态标签：**进行**（开发中）| **存档**（结项不迭代）| **更新**（存档重启）

### 4.3 版本管理结构

```
projects/{分类}/{项目名}-{状态}/
├── 项目索引.md          ← 版本一览+快速导航
├── v{版本号}-{描述}/     ← 版本文件夹
│   ├── 文件...
│   └── 改动说明.md
├── v{版本号}-{描述}/
└── 开发反思.md           ← 每次会话后自动追加
```

### 4.4 从 `_temp/` 移入的三套标准（组合使用）

| 条件 | 动作 |
|------|------|
| 你手动表示结项 | 分版本→移入 `projects/{分类}/{项目名}-存档/` |
| `_temp/项目/` ≥14天无改动 | 清理模式提示→你确认→移入 |
| 版本稳定后主动移入一个版本 | 版本文件夹移入，_temp/ 保留最新数据 |

## 五、`个性化/` 个人云盘管理规则

### 5.1 定位

`个性化/` 是你的个人数据资产中心。Hanako 通过此目录理解你的背景、状态和知识体系。不是冷冰冰的仓库，而是**动态更新的个人画像**。

### 5.2 资料/ — 文件存储

| 子目录 | 内容 | 规则 |
|--------|------|------|
| `备考/{科目}/` | 讲义/题本/笔记（地理教师编、法检书记员） | 只进不出，标记来源+适用阶段 |
| `通用参考/` | 技术文档、通用知识 | 只进不出 |
| `个人文档/` | 简历/证书/扫描件 | 只进不出 |
| `资料索引.md` | 标注收录状态 | 自动更新；不在本目录的标注位置 |

**资料索引.md 格式：**

```markdown
| 目录/文件 | 位置 | 内容 | 状态 |
|-----------|------|------|------|
| 备考/ | 个性化/资料/备考/ | 讲义题本 | ✅ |
| 服务器密码 | 仅存记忆 | 配置信息 | ⚠️ 未收录 |
```

### 5.3 事项/ — 有生命周期的"事"

命名格式：`{事项名}-{状态}`，状态标签同 projects/（进行/存档/更新）。

每个事项目录内的文件：

| 文件 | 作用 | 维护者 |
|------|------|--------|
| `事项索引.md` | 关联的资料/文件位置（链接到实际路径） | Hanako 更新 |
| `习惯文档.md` | 该事项的固定行为模式/规则 | Hanako 维护 |
| `工作流.md` | 具体操作步骤和规范 | Hanako 维护 |
| `备忘录.md` | 历史进度时间线 | 每次事项更新后追加 |

事项/ 中的开发事项（如"开发优化大师"）以索引和记录为主，实际代码仍在 `projects/` 中。

### 5.4 联动/ — 外部系统接入

记录你分散在多个平台数据的接入方式和联动流程。每个系统一个子目录：

```
个性化/联动/
├── 联动总索引.md         ← 所有系统概览
├── 思源笔记/              ← MCP+CLI，笔记读写
├── 记忆助手-MemOS/        ← CLI+API，知识库归档
└── 移动云盘/              ← 文件同步（预留）
```

每个系统内部四个文件：

| 文件 | 记录什么 |
|------|---------|
| `联动说明.md` | 连接方式（地址/端口/API/凭证）、笔记本/知识库 ID 映射 |
| `联动工作流.md` | Hanako 操作该系统的标准步骤和注意事项 |
| `内容索引.md` | 缓存外部系统中的文档结构和标签，联动时自动更新 |
| `联动记录.md` | 每次联动操作的时间、内容和结果 |

联动/ 的内容由 Hanako 在首次接入和每次联动后更新。

### 5.5 专项/ — 长久知识库

只增不减，无结束状态。分为四个领域：

| 领域 | 内容 | 来源 |
|------|------|------|
| **习惯** | 固定行为模式（每日复盘、反思节奏） | 从事项/和日常中提炼 |
| **技能** | 跨项目技术能力（反代开发、Go、New-API管理） | 从 projects/开发反思.md 中提炼 |
| **认知** | 思维模型、学习方法论、决策框架 | 从事项/和经验中提炼 |
| **领域** | 非技术长期知识（教育理论、法律基础） | 从备考/中沉淀 |

每个领域仅维护一个 `成长记录.md`：

```markdown
## 已沉淀
| 日期 | 条目 | 来源 | 状态 |
## 待沉淀
| 待整理内容 | 来源 | 备注 |
```

### 5.6 个性化/ 的维护规则

- **Hanako 自动维护**：日常聊天、操作、记忆和经验积累中，主动更新对应的索引、备忘录和成长记录
- **清理模式扫描**：遍历各子目录，重写索引和成长记录
- **不设自动删除**：资料/和专项/只进不出；事项/由状态标记管理

## 六、`data/` 冷区存储管理规则

### 6.1 定位

冷区。温度最低、访问频率最低、但保留最持久。东西一定是从其他目录冷却下来的。

```
data/
├── backups/     ← 服务器/数据库备份
├── archives/    ← 从 projects/ 清理的旧项目、_tools/闲置归档
└── 数据集/      ← >50MB 且不属 _tools/ 或 个性化/ 的数据
```

### 6.2 准入规则

| 准入 | 去向 | 禁止 | 理由 |
|------|------|------|------|
| 服务器/数据库备份 | backups/ | 可复用脚本 | 应进 _tools/ |
| 从 projects/ 清理的旧项目 | archives/ | 参考资料 | 应进 个性化/ |
| >50MB 的数据 | 数据集/ | 结项但能回顾的项目 | 留在 projects/存档 |
| _tools/闲置 >90天 | archives/ | — | — |

### 6.3 backups/ 管理

命名 `{来源}-{YYYYMMDD}-{描述}.{ext}`，保留最近 7 份，`备份索引.md` 记录使用历史。

### 6.4 archives/ 管理

每个归档包必含 `归档说明.md`（原始位置/内容/可激活条件）。`归档索引.md` 自动更新。

反向激活：你说"这个项目要继续"→ 从 archives/ 移回 `projects/{分类}/{项目名}-更新/`。

## 七、联动规则（闭环流转）

### 7.1 完整流转路径

```
对话产物 → _temp/对话/{主题}/
    ├── 截图/数据 → 有保留价值 → 建索引移入 个性化/资料/
    └── 脚本 → 可复用 → 移入 _tools/脚本/

开发项目 → _temp/项目/{项目名}/
    ├── 结项确认 → 分版本 → projects/{分类}/{项目名}-存档/
    └── 旧项目清理 → data/archives/

projects/存档/ → 你说继续 → projects/{分类}/{项目名}-更新/
data/archives/  → 你说激活 → projects/{分类}/{项目名}-更新/

_tools/闲置 → 确认 → data/archives/
data/archives/  → 仍有价值 → _tools/
```

### 7.2 个性化/ 的输入来源

```
记忆/经验 → 写入 个性化/专项/{领域}/成长记录.md
事项进展 → 追加 个性化/事项/{事项名}-{状态}/备忘录.md
工作流/习惯 → 更新 个性化/事项/{事项名}-{状态}/工作流.md、习惯文档.md
```
