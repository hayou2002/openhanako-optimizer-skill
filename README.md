# OpenHanako 优化大师 v2.0

> Hanako Agent 的「工作区系统治理」Skill。一站式管理工作区、缓存、技能、经验记忆、插件、开发、运行时，让杂乱变有序。

---

## 这个 Skill 解决什么问题

```yaml
问题一：工作区杂乱
  文件到处乱放，不知道哪些该留哪些该删，项目没有版本管理

问题二：经验与记忆分散
  工具经验、个人知识、项目反思分在 N 个地方，彼此不关联

问题三：清理靠手动
  缓存膨胀、日志堆积、旧对话附件占空间，没有自动机制

问题四：插件和技能管理混乱
  同名技能、废弃插件、孤儿目录，没有统一规范

问题五：开发无流程
  说干就干，没有需求拆解、提示词优化、测试、反思的闭环
```

---

## 优点

| 特性 | 说明 |
|------|------|
| 🎯 一站式 | 说一句「优化大师」，覆盖 9 大板块 |
| 🧩 选单式 | 选一个做一项，做完自动回菜单，不轰炸 |
| 🤖 自动化 | 黄色项自动执行，红色项等你确认 |
| 🧠 自沉淀 | 清理完自动提炼经验到知识库 |
| 🔌 可适配 | 通用规则开箱即用，个性化规则自动检测 |
| 🔰 零门槛 | 说「初始化」自动建立所有目录和文档 |

## 快速开始

### 安装

将整个 `openhanako-优化大师-v2.0/` 目录安装为 Hanako Skill：

```bash
# 通过 install_skill 安装
install_skill --local_path openhanako-优化大师-v2.0
```

### 首次使用

安装后说一次「初始化」，会自动建立标准目录结构、参考文档和模板文件。

```
你：初始化
我：正在建立目录结构……正在写入参考文档……初始化完成
```

### 日常使用

```
你：优化大师
我：展示菜单（9 大板块 + 初始化）

你：① 工作区   → 扫工作区 → 出报告 → 确认执行 → 回菜单
你：② 缓存     → 扫缓存 → 出报告 → 确认执行 → 回菜单
   ……循环

你：全部       → 一次扫全部板块
你：退出       → 结束
```

### 一句话总结

**说你遇到的任何问题 → 我出菜单 → 你选 → 我扫 → 你确认 → 我执行。**

## 架构：从宏观到微观

### 宏观层：温度体系

工作区按访问频率划分为五层，每层有独立的管理策略：

```
🔥 _temp/     热区   临时缓冲区，7~30天流转
☀️ _tools/    暖区   高频复用工具箱，严格准入
🌤 projects/  常温区 分版本项目管理，状态标记
☀️ 个性化/    暖区   个人云盘 + 事项 + 知识库
❄️ data/      冷区   备份 + 归档，从别处冷却
```

### 中观层：三大循环

```
日常清理循环                      开发循环                     知识循环
┌─────────────┐              ┌─────────────┐              ┌─────────────┐
│ 说优化大师   │              │ 我想做xx    │              │ 工具失败     │
│ →选项目     │              │ →需求拆解   │              │ →自动记录    │
│ →扫描       │              │ →提示词优化 │              │ →清理时提炼  │
│ →确认执行   │              │ →生成       │              │ →存入知识库  │
│ →回菜单     │              │ →测试       │              │             │
└─────────────┘              │ →发布反思   │              └─────────────┘
                              └─────────────┘
```

### 微观层：9 大板块

| # | 板块 | 核心逻辑 | 生命周期 |
|---|------|---------|---------|
| ① | 工作区 | 标准目录 + 白名单 + 项目规范 | 持续维护 |
| ② | 缓存 | 日志3天、检查点10个 | 到阈值清理 |
| ③ | 技能 | 同名检测 + bundle匹配 | 安装时检查 |
| ④ | 经验与记忆 | 三模协作（自动/辅助/个人） | 持续积累 |
| ⑤ | 会话 | 空目录 + 大文件预警 | 到阈值清理 |
| ⑥ | 插件 | 热重载 + 孤儿目录 + 补丁 | 运行时维护 |
| ⑦ | 配置 | 设置合理性检查 | 按需 |
| ⑧ | 开发 | 六步流程 + 临时文件分层 | 开发周期 |
| ⑨ | 运行时 | 归档对话分层 + 缓存总容量 | 到阈值清理 |

### 初始化层：第⑩板块

```
一次初始化，永久受益：
  标准目录 → 参考文档 → 模板文件 → 适配检测
  之后每次使用，只扫增量变化
```
