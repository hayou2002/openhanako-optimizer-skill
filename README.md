# OpenHanako 优化大师 v2.2

> Hanako Agent Skill：一站式工作区治理工具。让你的工作区从"知道该整理"到"自动保持整洁"。

## 双通道执行

优化大师提供两条互补的执行通道：

| 通道 | 触发方式 | 适用场景 |
|------|----------|----------|
| 🔵 **手动** | 说「优化大师」出菜单选择板块 | 结构变化时做全盘扫描 / 临时清理 |
| 🔵 **自动化** | 定时任务后台静默执行 | 每日归类 + 每 2 天经验沉淀 |

两条通道共享同一套规则手册，规则更新自动同步。

### 快速开始

```bash
# 在 HanaAgent 中安装
install_skill --github_url https://github.com/hayou2002/openhanako-optimizer-skill
```

说「优化大师」展开菜单，先跑一次「初始化」建立骨架，然后「全部」全盘扫描。之后日常使用**手动通道**或**定时自动化**皆可。

## 菜单

```
⚙ 优化大师 v2.2 — 请选择：

  说「初始化」       首次建立目录、规则文档 + 设置定时任务
  说「工作文件夹」   归类文件 + 扫描缓存、会话、临时文件 + 关联同步
  说「插件技能」     扫描技能 / 插件 / 开发规范
  说「系统运行」     扫描经验与记忆 / 配置 / 运行时缓存
  说「全部」        一次性扫描所有
  说「任务状态」     查看定时自动化任务运行状态
  说「退出」        结束
```

### 标准交互流程

```
说板块名 → 扫描 → 报告 → 确认执行 → 沉淀（含关联同步）→ 连贯询问
```

### 确认规则

- **低风险**（移动 .md、删除临时文件）：汇总执行，不逐条确认
- **中风险**（移动数据文件、归档项目）：分组建议，批量确认
- **高风险**（删除 .exe、卸载插件）：逐条确认
- 随时说「中断」停止

## 设计原理

### 架构层次

```
┌─────────────────────────────────────────────────┐
│  第一层：触发与入口（双通道）                     │
│  手动：「优化大师」→ 菜单 → 确认 → 执行           │
│  自动：定时任务 → 后台静默归类 + 关联同步           │
├─────────────────────────────────────────────────┤
│  第二层：执行引擎（SKILL.md）                     │
│  定义板块划分、扫描规则、交互流程、确认逻辑        │
├─────────────────────────────────────────────────┤
│  第三层：规则手册（优化规则手册.md）               │
│  11 条规则分类，覆盖工作区/缓存/技能/插件/经验/    │
│  关联同步/技能文件存储                            │
├─────────────────────────────────────────────────┤
│  第四层：规范文档与参考                           │
│  文件组织规范.md / 笔记排版语法规则 / 重点笔记规则 │
├─────────────────────────────────────────────────┤
│  第五层：运行态（检查点/留痕/适配检测）            │
│  _temp/检查点/ 记录每次操作，适配检测感知本地环境  │
└─────────────────────────────────────────────────┘
```

每一层只依赖下层，不跨层调用。规则手册（第三层）是唯一主规则源，手动和自动化通道都引用它。

### 文件流闭循环

```
对话产物 → _temp/对话/{主题}/
    ├── 有保留价值 → 索引移入 个性化/资料/
    └── 可复用脚本 → 移入 _tools/脚本/

开发项目 → _temp/项目/{项目名}/
    ├── 结项 → 移入 projects/{分类}/{项目名}-存档/
    └── 超 14 天无改动 → 提示结项

_tools/闲置 >90 天 → 提示归档 data/archives/
data/archives/ → 你说继续 → 反向激活回 projects/
```

### 温度体系

| 温度 | 目录 | 清理策略 |
|------|------|---------|
| 🔥 热 | `_temp/` | 7~30 天超期自动提醒流转 |
| ☀️ 暖 | `_tools/`、`个性化/` | 90 天闲置预警，不自动删 |
| 🌤 常温 | `projects/` | 结项移入存档 |
| ❄️ 冷 | `data/` | 备份保留 7 份，可反向激活 |
